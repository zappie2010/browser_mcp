# 03. Browser Controls 실습

> **소요 시간**: 약 30분  
> **학습 목표**: Cursor Browser Controls를 사용하여 웹 애플리케이션의 다양한 기능을 수정하고 테스트

---

## 실습 전 준비사항

### 1. 서버 실행 확인
```bash
# 터미널에서 실행
python app.py
```

브라우저에서 `http://localhost:5000`에 접속하여 페이지가 로드되는지 확인하세요.

### 2. Cursor Browser Controls 확인
Cursor Chat에서 다음 명령을 실행:
```
브라우저를 열어서 http://localhost:5000에 접속해줘
```

---

## 실습 1: CSS 스타일 변경 (5분)

### 목표
헤더와 버튼의 색상 및 스타일을 변경하여 Browser Controls의 실시간 피드백 기능 체험

### 변경 요구사항
1. 헤더 배경색을 빨간색 그라데이션으로 변경
2. 스크래핑 버튼을 더 크고 눈에 띄게 수정
3. 변경사항을 브라우저에서 즉시 확인

### Cursor Chat 프롬프트

**프롬프트 1**: 헤더 색상 변경
```
헤더의 배경색을 빨간색 그라데이션으로 변경해줘. 
기존 파란색 대신 #ff6b6b에서 #ee5a6f로 변경하고, 
브라우저에서 확인해줘.
```

**예상 결과**:
- `static/style.css` 파일의 `.header` 스타일 수정
- 브라우저 자동 새로고침
- 스크린샷으로 결과 확인

**프롬프트 2**: 버튼 크기 및 스타일 변경
```
스크래핑 버튼을 더 크게 만들어줘.
padding을 20px 40px로 변경하고,
그림자 효과를 추가해서 더 눈에 띄게 만들어줘.
```

**예상 결과**:
- `.scrape-btn` 클래스의 padding 변경
- box-shadow 추가
- 브라우저에서 실시간 확인

### 직접 해보기
다음 스타일도 변경해보세요:
- 결과 카드의 테두리 색상
- 푸터의 배경색
- 테이블의 행 간격

## 실습 2: 링크 필터링 기능 추가 (7분)

### 목표
스크래핑 결과의 링크 목록을 필터링할 수 있는 검색 기능 추가

### 변경 요구사항
1. 링크 목록 위에 검색 입력 필드 추가
2. 입력한 키워드로 링크 필터링
3. 필터링된 결과만 표시

### Cursor Chat 프롬프트

**프롬프트 1**: HTML 구조 추가
```
스크래핑 결과의 링크 섹션에 검색 입력 필드를 추가해줘.
사용자가 키워드를 입력하면 링크 목록을 필터링할 수 있도록 해줘.
```

**프롬프트 2**: JavaScript 기능 구현
```
링크 검색 입력 필드에 입력할 때마다 실시간으로 링크 목록을 필터링해줘.
대소문자를 구분하지 않고, 링크 텍스트와 URL 모두에서 검색되도록 해줘.
```

**프롬프트 3**: 스타일링
```
링크 검색 입력 필드의 스타일을 URL 입력 필드와 일관되게 만들어줘.
```

### 테스트 시나리오
1. `https://example.com` 스크래핑
2. 링크 검색 필드에 "more" 입력
3. "Learn more" 링크만 표시되는지 확인

## 실습 3: 로딩 애니메이션 개선 (5분)

### 목표
스크래핑 중일 때 더 명확한 로딩 표시

### 변경 요구사항
1. 로딩 스피너 애니메이션 추가
2. 버튼 비활성화 시 스타일 변경
3. 로딩 중 메시지 개선

### Cursor Chat 프롬프트

**프롬프트 1**: 로딩 스피너 추가
```
스크래핑 버튼에 로딩 스피너 애니메이션을 추가해줘.
스크래핑 중일 때 회전하는 아이콘이 표시되도록 해줘.
```

**프롬프트 2**: 버튼 상태 개선
```
스크래핑 중일 때 버튼의 배경색을 회색으로 변경하고,
커서를 not-allowed로 표시해줘.
```

### 테스트 시나리오
1. URL 입력
2. 스크래핑 시작 버튼 클릭
3. 로딩 스피너가 표시되는지 확인
4. 버튼이 비활성화되는지 확인

---

## 실습 4: 에러 처리 개선 (5분)

### 목표
더 친절하고 명확한 에러 메시지 표시

### 변경 요구사항
1. 에러 메시지에 아이콘 추가
2. 재시도 버튼 추가
3. 에러 유형별로 다른 색상 표시

### Cursor Chat 프롬프트

**프롬프트 1**: 에러 메시지 개선
```
에러 메시지 영역에 적절한 아이콘을 추가하고,
재시도 버튼을 추가해줘. 재시도 버튼을 클릭하면
이전에 입력한 URL로 다시 스크래핑을 시도하도록 해줘.
```

**프롬프트 2**: 에러 유형별 스타일
```
네트워크 오류는 주황색, 
URL 형식 오류는 빨간색,
타임아웃 오류는 노란색으로 표시되도록 해줘.
```

### 테스트 시나리오
1. 잘못된 URL 입력 (`invalid-url`)
2. 에러 메시지 확인
3. 재시도 버튼 클릭 테스트

---

## 실습 5: 결과 데이터 내보내기 (5분)

### 목표
스크래핑 결과를 JSON 파일로 다운로드

### 변경 요구사항
1. 결과 영역에 "JSON으로 내보내기" 버튼 추가
2. 버튼 클릭 시 JSON 파일 다운로드
3. 파일명에 타임스탬프 포함

### Cursor Chat 프롬프트

**프롬프트**: 데이터 내보내기 기능
```
스크래핑 결과 영역에 "JSON으로 내보내기" 버튼을 추가해줘.
버튼을 클릭하면 스크래핑 결과를 JSON 파일로 다운로드할 수 있도록 해줘.
파일명은 "scrape-result-YYYYMMDD-HHMMSS.json" 형식으로 만들어줘.
```

### 테스트 시나리오
1. 스크래핑 실행 후 결과 확인
2. "JSON으로 내보내기" 버튼 클릭
3. JSON 파일 다운로드 확인
4. 파일 내용 검증

## 실습 6: 다크모드 추가 (8분)

### 목표
다크모드 토글 기능을 추가하여 CSS 변수 활용 체험

### 변경 요구사항
1. 헤더에 다크모드 토글 버튼 추가
2. 다크모드용 CSS 변수 정의
3. 토글 시 전체 색상 테마 전환
4. 사용자 선택 저장 (localStorage)

### Cursor Chat 프롬프트

**프롬프트 1**: 다크모드 CSS 변수
```
다크모드를 지원하도록 CSS 변수를 추가해줘.
:root에 라이트 모드 변수가 있고,
[data-theme="dark"]에 다크 모드 변수를 정의해줘.
배경은 어두운 회색, 텍스트는 밝은 색으로 설정해줘.
```

**프롬프트 2**: 토글 버튼 추가
```
헤더 오른쪽 상단에 다크모드 토글 버튼을 추가해줘.
버튼에는 해/달 아이콘을 사용하고,
클릭하면 테마가 전환되도록 해줘.
```

**프롬프트 3**: localStorage 저장
```
다크모드 설정을 localStorage에 저장해서
페이지를 새로고침해도 설정이 유지되도록 해줘.
```

### 테스트 시나리오
1. 다크모드 토글 버튼 클릭
2. 전체 색상 테마 변경 확인
3. 페이지 새로고침 후 설정 유지 확인
4. 브라우저 개발자 도구에서 localStorage 확인
